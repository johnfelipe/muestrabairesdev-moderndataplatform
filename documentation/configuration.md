# `modern-data-platform` - Configuración v1.16.0

Esta es la documentación de los valores de configuración que se pueden sobrescribir mediante un archivo YAML personalizado. Todos los valores predeterminados se definen en [`../modern-data-platform-stack/generator-config/vars/config.yml`](../modern-data-platform-stack/generator-config/vars/config.yml).

## Configuración general

Hay algunas configuraciones generales que controlarán el comportamiento de todos o un grupo de servicios. Estos se enumeran en la tabla a continuación.

| Configuración | | predeterminada Desde | Descripción |
|------------------------------------------------	|:-------:	|-------	|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------	|
| `use_timezone`                             	|  	| 1.5.0 | La zona horaria que se va a usar para toda la pila. De forma predeterminada está vacía, por lo que la zona horaria del motor de Docker no se cambia y se ejecutará como `Etc/UTC`. Si desea establecerlo en otra zona horaria, especifique una cadena de zona horaria de Unix, como `Europe/Zurich` o `America/New_York`. Puede encontrar una descripción general de las zonas horarias válidas aquí: <https://en.wikipedia.org/wiki/List_of_tz_database_time_zones> |\
| `private_docker_repository_name`                             	| `trivadis` 	| 1.5.0 | Las imágenes de Docker que no estén disponibles en Docker Hub público se recuperarán mediante este repositorio privado. Por defecto apunta a `trivadis` y debe iniciar sesión primero, antes de poder usar la pila generada, si ha seleccionado una imagen privada. Utilice esta configuración para apuntar a su propio registro privado de Docker si es necesario. |\
| `uid`                             	| `1000` 	| 1.9.0 | El UID que se va a usar cuando se usa la propiedad "user" en un servicio para invalidar al usuario dentro del contenedor. |\
| `data_centers`                             	| `dc1,dc2` 	| 1.14.0 | Una lista separada por comas de nombres de centros de datos, para usar si la propiedad `data_center_to_use` tiene un valor != 0. |\
| `data_center_to_use`                             	| `0` 	| 1.14.0 | El centro de datos que se va a utilizar, si se deben simular varios CC para una configuración de Kafka. |\
| `copy_cookbook_data_folder`                             	| `true` 	| 1.14.0 | Copia todos los `data` carpetas de las diversas recetas de libros de cocina en el `data-transfer/cookbook-data` carpeta. |

## Servicios de plataforma

Las opciones de configuración para habilitar/deshabilitar un servicio determinado se denominan `XXXXX_enable` donde XXXXX es el nombre del servicio (solía ser nombrado `XXXXX_enabled` en la versión 1.0.0).
Para cada servicio puede haber algunas otras configuraciones, como controlar el número de nodos con los que iniciar el servicio, si el servicio debe asignar un volumen de datos en el contenedor o controlar algunas otras propiedades de configuración propietarias del propio servicio.
